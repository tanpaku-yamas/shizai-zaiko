データ管理

発注推奨リスト

コピーしてイントラの発注画面に貼り付けるか、メールで自分宛に送れます。推奨数 = 引当後の残在庫を発注点(20)まで戻す数量です。

基本の使い方

1. 週次の運用フロー
  1. イントラから説明会計画のExcelをダウンロード
  2. 予定タブにドロップ(またはタップして選択)
  3. ヘッダーの表示を確認 — 「在庫OK」なら何もしなくてよい
  4. 「不足○品目」なら発注タブへ → コピーしてイントラ発注、またはメール送信
  5. 資材が届いたら在庫タブで数量を増やす(入庫)
  6. 説明会を実施したら予定を開いて「実施済みにする」 → 在庫が自動で減る
手入力はこの「入庫」と「実施済みチェック」の2つだけです。
2. 予定タブの見方
各行の右端は予定人数。NM/G4チップはH列(主品目)からの自動判定です。行をタップすると、その説明会に必要な資材の内訳が開きます。赤枠+「資材不足」は、その説明会で使う資材のどれかが発注点割れしているサインです。
取り込みのたびに未実施の予定は最新のExcel内容で置き換えられます(実施済みの記録は保持)。今日より前の行は取り込み時にスキップされます。
3. 在庫ゲージの見方
濃い色=引当後に残る在庫、斜線=今後の説明会に引き当てられている分、縦線=発注点。濃い色が赤い時は、引当後の残りが発注点を下回っています。
4. 品目の追加・削除・名前変更
追加: 在庫タブの「＋品目を追加」。追加品目は説明会引当の対象外で、発注点を切ったらアラートに載ります(文具向け)。
名前変更: 品目名をタップしてそのまま書き換え(イントラの正式名称に合わせるのを推奨)。
削除: 品目を左にスワイプ(PCは右クリック)→ 赤い削除ボタン。初期の資材を削除すると説明会引当の計算からも外れます。廃番になった時だけ使ってください。
5. 発注点の変え方
各品目の「発注点」の数値欄を直接編集。アラート判定・ゲージの縦線・発注推奨数(発注点まで戻す数量)がすべて連動します。初期値は全品目20です。

よくあるQ&A

データはどこに保存される?
この端末のブラウザ内(localStorage)だけです。外部サーバーには一切送信されません。裏返すと、PCとスマホでデータは同期されません。どちらか一方を正として運用してください。
データが消えることはある?
ブラウザの「閲覧データ削除」で消えます。iPhoneのSafariは長期間開かないと自動削除されることもあります。対策として月1回(棚卸し時)のバックアップ保存を推奨。消えてもJSONから「復元」ですぐ戻せます。
チップが「要確認」になる
H列の主品目から NM / G4 のどちらか判定できなかった行です。共通資材(クリアファイル、バッグ/封筒)だけ引き当てられ、資材7種は引当されません。イントラ側のH列の値を確認してください。
「過去・未処理」と出ている予定は?
日付が過ぎたのに実施済みチェックされていない予定です。引当からは自動で外れていますが、在庫の減算がまだなので、実施したなら「実施済みにする」、中止なら削除してください。ここを放置すると在庫数が実態とズレます。
実在庫と数字が合わない
在庫タブで実数に直接書き換えてください(数値欄をタップ)。ズレの主因は①実施済みチェック漏れ ②説明会以外での資材使用(単発の面談持参など)。②が多いなら、使った分をその都度−ボタンで引くのを習慣に。
バッグと封筒の「合算」って?
説明会1回につきどちらかを人数分持参すればよいため、2品目の在庫合計で足りるかを判定しています。合算の発注点は上に表示される側(ファイルdeバッグ)の値を使用。実施済み処理ではバッグから優先して減算します。
発注推奨数の計算式は?
推奨数 = 今後の説明会の引当合計 + 発注点 − 現在庫。つまり「全予定を消化しても発注点分の在庫が手元に残る」数量です。
間違えて実施済みにした/削除した
実施済みは「実施済み(○件)」を開いて取消で在庫ごと元に戻ります。品目や予定の削除は取消せないので、バックアップからの復元で戻してください。
メール送信ボタンの仕組みは?
メールアプリが宛先・件名・本文入りで起動する方式です(自動送信ではなく、送信ボタンは自分で押します)。宛先は初回に入力すれば保存されます。
データはこの端末のブラウザ内(localStorage)にのみ保存されます。端末変更時はバックアップ→復元を使用してください。
